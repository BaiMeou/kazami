## 格式规范
- 正文不加 Markdown 标头（`#`、`##`），直接输出纯文学段落
- 对话句尾不加句号（`。"` → `"`），问号和感叹号保留
- 段尾必须有终止符，描述段和动作段以句号收束
- 省略号表示语气断续和停顿
- 场景描写和对话必须分开，不能揉在一起
- 所有文件名和目录名使用纯中文
- 知识库文件不加 YAML frontmatter，纯内容高信息密度
- 索引文件放在 `索引/` 文件夹
- NSFW番外开头保留 `> **简介**：...` 和 `> **关键词**：...` 的 blockquote 格式
- 重写时需一并审查简介、关键词、文件名

## 写作禁令

### 硬禁止（零容忍）
- **禁止词汇**：共犯、该死、见鬼、小兽、极其、四肢百骸、生理性、虔诚、不容质疑、死死、不易察觉、指节泛白、颧骨、黏腻
- **禁止句式**："不是……是……"/"不是……而是……"纠正性句式，直接陈述
- **禁止脸部套路**：眼神冰冷/深邃/暗了暗、眸色一沉、眉头微皱、嘴角勾起、邪魅一笑、似笑非笑
- **禁止身体套路**：指节泛白、青筋暴起、呼吸一滞、倒吸一口凉气、喉结微滚、浑身一震、身子一僵
- **禁止描写手法**：带"语气"前缀的形容词、蚊子哼哼、小动物比喻、手术刀比喻、针刺类比喻
- **禁止情绪贴标签**：情绪必须通过动作、错误和环境互动演出来

### 写作留意（不是禁令，写的时候留个心眼）
- 以下词汇容易被滥用，偶尔出现完全没问题，密集堆叠才需要注意：
  - 破折号（——）只在合适场景出现，不滥用
  - 一丝、泛白、禁忌、狡黠、呜咽、搔刮、压抑、嘶吼、野兽般、小猫爪子、涟漪、几不可查/几不可闻、圣物、信徒、宛若神明、不容抗拒、献祭、仪式、沙哑、温热、灭顶、古井、巨浪、审判、征服、毁天灭地、细若蚊呐、慵懒、嘶哑、磁性、深沉、支配、占有、乐章、凝固、戏谑、侵略性、深渊、仿佛、陷入、如同、就像、勾起、嘴角什么什么弧度、若有若无、心湖、崇拜、狂热、粗暴、画圈、研磨、氤氲、提线木偶、揉入骨血、痉挛、低笑、无意识的勾住、裹挟着糖、细密的汗珠、小猫、微哑、无奈又宠溺
- "极X"句式（极轻、极细、极淡等）多数情况下"微微""轻轻""很淡"更自然，不需要禁止但写的时候想一想
- 头发遮住表情：注意物理视角是否成立，背对着的人看不到正面
- 微微/轻轻等词每章6-10个是合理的，不要机械削减
- NSFW亲密场景中部分词汇可合理使用

## 文学质量
- 追求高文学性，不水字数，阅读流畅是第一优先级
- 偏好连贯长句驱动，大段落有分量感和呼吸感，段落长短交替有呼吸感
- 短句做节拍和场景转场（"她蹲了下去。""第二天。"）
- 避免量化数据（距离、温度、时间等数字），用模糊感官词（漫长/瞬息/愈发）替代
- 对话要自然：断续、口癖、停顿、语气词
- 比喻不滥用，一个段落里不超过一个，删掉后句子依然成立就删掉
- 内心独白自然编织进叙事中，不单独标出
- 减少尾巴/耳朵当情绪标签的泛滥，仅在关键情绪转折时使用。晓光九条尾巴平时压在一起很难活动，日常尾巴互动可以写（抱着当抱枕、被压麻了），但"尾巴兴奋地摇来摇去"这类情绪指示器不要写

## 物理常识（描写必须符合现实物理）
- **身体结构**：被占用的部位不能同时做别的事；体位必须物理可行（后入式做不到面对面的动作，背对时双腿夹不到腰）；背对着的人看不到正面表情
- **衣物不会轻易滑落**，不要随便写衣物自己滑下来的情节
- **重力与垂坠**：悬空部分（裙摆、袖口、发梢）自然下垂不会莫名上卷；厚重面料笔直下垂折痕宽大，轻薄面料随身体起伏堆叠折痕细碎
- **动态与惯性**：角色改变姿势时衣物方向随之改变；急停或转身时衣摆、头发、挂饰因惯性继续运动；重裙摆摆幅小且滞后，轻纱飘逸跟随性强
- **张力与压缩**：动作导致衣物受力点转移；紧身衣被身体撑开会变薄透光；弹性材质在弯折处产生褶皱和空隙；衣物对活动有限制或顺应
- **环境交互**：气流扰动轻量物体（发丝、裙边、飘带）；光线在不同曲面上产生高光移动和阴影深浅变化
- **流体交互**：不同面料吸水速度和渗透扩散不同；湿透的轻薄面料因水的表面张力吸附皮肤呈半透明；液体流经多层衣物结构（腰带、领口、袜口）时路径会改变——被阻挡积聚、顺边缘滴落或渗透至下层
- **体温传导**：任何液体停留在皮肤上久了都会被体温传导变成微暖的

## 写作实战步骤

**正文写作**：读剧情大纲和时间线 → 读对应卷剧情 → 读涉及角色档案 → 读相关世界观扩展档案 → 动笔（移除标头、角色穿插全文不突然出现、结局留开放空间、上下文连贯）→ 自查

**番外重写**：读原篇（完整逐字）→ 读涉及角色档案 → 写 → 自查（角色是否走形、段尾是否悬空、引号内是否误加句号、亲密场景白霜是否误用禁用词）

**角色/剧情重构**：读旧档案 → 用选项让用户逐项确认 → 写新档案 → 用户审批
