# 银月脉纪（kazami）项目写作约束

## 格式规范
- 正文不加 Markdown 标头（如 `#`、`##`），直接输出纯文学段落
- 重写时需一并审查并修改简介、关键词、文件名
- NSFW番外开头保留 `> **简介**：...` 和 `> **关键词**：...` 的 blockquote 格式
- 对话句尾不加句号（`。"` → `"`），问号和感叹号保留
- 所有文件名和目录名使用纯中文
- 知识库文件（角色/世界观/剧情/）不加 YAML frontmatter，纯内容高信息密度，利于 RAG 检索
- 索引文件放在 `索引/` 文件夹，不在知识库文件内做索引

## 标点实战规范（通过反复试错验证）

1. **段尾必须有终止符**：描述段和动作段以句号收束，不悬空。每个段落是一个完整的语义单元。
2. **对话引号内句尾不加句号**：`。"` → `"`，问号和感叹号保留（`？"` `！"` 不动）。
3. **禁止使用破折号（——）**：用逗号、句号或重写句子结构来替代。
4. **省略号用来表示语气断续和停顿**：对话中的犹豫、思考中的留白。
5. **场景描写和对话必须分开**：描写段落是描写段落，对话独立成行，不能揉在一起。

## 文学质量
- 追求高文学性，不水字数
- 阅读必须通顺流畅，这是第一优先级
- 清理滥用的破折号
- 避免量化数据（具体距离、温度、时间等数字），改用感性化描写传达同样的信息，保持沉浸式阅读体验
- 像有功底的中文小说作者，段落长短交替有呼吸感。短句做节拍和场景转场（"她蹲了下去。" "第二天。"）
- 对话要自然：断续、口癖、停顿、语气词等等
- 偏好长句子和大段落，段落要有分量感和呼吸感
- 减少尾巴/耳朵等肢体动作描写的泛滥，仅在关键情绪转折时使用。晓光九条尾巴平时压在一起很难活动，不要写成会根据情绪摇/竖/炸毛的情绪指示器。日常的尾巴互动可以写（比如抱着尾巴当抱枕、尾巴被压麻了换个姿势），但"尾巴兴奋地摇来摇去""尾巴竖起来"这类当情绪标签用的不要写
- 内心独白自然编织进叙事中，不单独标出

## 重写方法
- 必须逐篇完整阅读后再改，不能用 grep 搜索批量处理
- 小说改动采用整章重写，非段状重写或机械替换
- 每章目标 300+ 行
- 角色档案必须在小说重写之前完成（重写要依据档案）
- 新角色/势力引入必须穿插全文，不能某章突然出现然后消失
- 结局留开放空间给续写（故事止于晓光14岁）

## NSFW番外规则
- 每篇字数无上限，不设硬性限制
- 必须有日常铺垫 → 自然过渡 → 亲密场景
- 可写角色（外观年幼）：晓光、白霜、澄夜、红绡、绒羽、铃、霜月
- 不写角色（外观成熟）：青栀、紬、九阙、千翼、静
- 判断标准是外观年龄而非实际年龄
- B杯角色也可以写乳交
- 类型覆盖：solo、百合、乳交、3P,口等，每篇都要有性交环节，如果是百合则指交

## 重写实战步骤

**角色设定重构**：读旧档案 → 用选项让用户逐项确认关键设定（性格核心、能力、与主角关系、在剧情中的功能） → 写新档案 → 用户审批。

**番外重写**：读原篇（完整逐字）→ 读涉及角色档案 → 拆解范文结构 → 写 → 自查（核对角色是否走形、段尾是否悬空、引号内是否误加句号、破折号是否合理、亲密场景白霜是否误用禁用词）。

**正文重写**：读重构后的剧情大纲和时间线 → 读对应卷剧情 → 读涉及角色档案 → 读相关世界观扩展档案 → 动笔（移除标头、300+行/章、角色穿插全文不突然出现、结局留开放空间、上下文必须连贯）→ 自查。
